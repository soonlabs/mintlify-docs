# Usage & Tokens

Every AI response consumes tokens based on input (user message + context) and output (agent response) length. Monitor your balance on the **Billing** page.

## Understanding Tokens

- **1 token** ≈ 4 English characters or 0.75 words
- A typical conversational exchange uses 500–2,000 tokens
- Knowledge base lookups add context tokens per request
- Longer persona descriptions increase per-request consumption

## Usage Dashboard

The **Billing** page shows:

| Metric | Description |
|--------|-------------|
| **Tokens Used** | Total tokens consumed (priced in USD) |
| **Remaining Credits** | Starter credits + top-up balance |
| **Usage Trend** | Daily token consumption chart |
| **Top-up Packs** | Available credit packs to purchase |

## When Credits Run Out

The bot pauses responding. Buy any **top-up pack** from the Billing page — your bot resumes immediately after checkout.

<Warning>**Top-up credits never expire** — buy once, use whenever. No reset clocks to worry about.</Warning>

## Reducing Token Consumption

- High-volume groups should pick the **Economy tier** (`gpt-4.1-nano` / `gpt-4o-mini`) — up to **90%** cheaper than Premium while still producing solid responses
- Put FAQs in the **knowledge base** rather than bloating the persona prompt — the knowledge base is retrieved on demand; the prompt ships with every request
- Write specific, concrete moderation rule descriptions to reduce inference overhead

## Billing History

**Billing → History** lists all purchases (bot creation + top-up packs) with timestamp and amount.

<Note>**No formal invoices / tax receipts are issued at this time.** If your accounting flow requires documentation, a screenshot of the History page can serve as proof of payment, or contact support for clarification.</Note>
