# 人设配置

**人设（Persona）** 决定 Bot 的语气、说话风格和行为方式。NebulaClaw 预置了四种社群定位预设，也支持完全自定义。

## 四种社群预设

| 预设 | 适合社群 | 调性要点 |
|------|---------|---------|
| **项目社群** | Web3 项目方、产品社区 | 专业、有耐心、围绕项目文档 + 反 shill |
| **KOL / Alpha 群** | 付费 Alpha / 研究群 | 中立、高信噪比、DYOR 提醒、过滤广告 |
| **交易员社群** | 交易讨论群 | 市场中立、不做投资建议、提示风险 |
| **Meme 社群** | Degen / Meme 币群 | 高能量、玩梗、但严查诈骗链接和假空投 |

每个预设都预置了**对应头像**和经调优的 Persona Prompt，点击即用，右侧聊天预览会实时更新。

## 自定义人设

选择 **Custom** 可直接编辑 Persona Prompt。常见自定义维度：

- Bot 的说话方式（语气、正式程度、语言偏好）
- 需要强调或回避的话题
- 项目专属的引导（例如"遇到质押问题引导到 /staking"）
- 特定角色设定或背景故事

**示例（在"项目社群"预设基础上追加）**：

```
你是 [项目名] 的官方社区管家。
- 遇到"怎么质押"类问题，引导用户去 /staking 文档
- 不讨论币价、不做投资建议
- 遇到带"保底翻倍"、"返利"的消息主动提醒诈骗风险
- 保持专业友好的语气
```

<Tip>切换到其他预设时**自定义文本会被保留**，可以放心比较不同预设效果后再切回 Custom。</Tip>

## 人设 vs 知识库——该写在哪？

| 写在 Persona | 写在知识库 |
|------------|-----------|
| **怎么说**——语气、态度、禁忌话题 | **说什么**——具体的 FAQ、产品信息、规则 |
| 高频、跨对话稳定 | 按需检索、内容多 |
| 长度应短（每次请求都会携带） | 可以放很多（按需命中） |

<Warning>Persona Prompt 太长会导致每次响应的 Token 开销变高。**具体知识尽量塞到知识库，不要堆在 Persona 里**。</Warning>

## 保存更改

修改后页面底部会出现 **Save** 按钮，保存即生效，无需重启 Bot。
