# 用量与 Token

Bot 的**三类核心行为**都会消耗 Token：

| 行为 | 触发条件 | 使用模型 |
|------|---------|---------|
| **AI 回复** | 用户在群内发消息、@Bot 提问 | 你选择的模型（经济 / 均衡 / 旗舰） |
| **违规检测** | 每条群消息自动扫描是否触发审核规则 | 系统内置 `gpt-4o-mini`（固定） |
| **情绪分析** | 每条群消息自动判断 sentiment | 系统内置 `gpt-4o-mini`（固定） |

<Tip>违规检测和情绪分析使用低成本的 `gpt-4o-mini` 固定运行，**不受你选择的模型档位影响**——即使你选了旗舰档，这两项的消耗依然很低。主要的 Token 开销来自 **AI 回复**。</Tip>

## 理解 Token

- **1 个 Token** ≈ 4 个英文字符 / 1-2 个中文字
- 一次典型的对话交互消耗 500–2,000 Token
- 知识库检索会增加每次请求的上下文 Token
- Persona Prompt 越长，每次请求的基础消耗越高

## 用量面板

**Billing** 页面显示：

| 指标 | 说明 |
|------|------|
| **已用 Token** | 已消耗的 Token 总量（美元计价） |
| **剩余额度** | 当前月度额度余额 + 充值包余额 |
| **用量趋势** | 每日 Token 消耗图表 |
| **充值包** | 可购买的额度面额列表 |

## 额度用尽后

Bot 会暂停响应。在 **Billing** 页点击任意**充值包**完成支付即可立即恢复。

<Warning>**月度 $5 额度每个计费周期结束时清零**——没用完不累积。**充值包额度永不过期**，用多少算多少。</Warning>

## 降低 Token 消耗

- 高消息量群组选用 **Economy 档**（`gpt-4.1-nano` / `gpt-4o-mini`），相比 Premium 档可降低 Token 消耗最多 **90%**，同时保持良好回复质量
- 把常见问题写进**知识库**而不是塞进 Persona Prompt——知识库按需检索，Prompt 每次都会全量携带
- 在审核规则里使用具体描述，减少 AI 推理开销

## 计费历史

在 **Billing → History** 查看历次购买记录（Bot 购买 + 充值包 + 时间 + 金额）。

<Note>**当前版本不提供正式发票（Invoice / Tax Receipt）。** 如果你的财务流程需要凭证，可截图"History"页的购买记录作为付款证明，或联系支持团队确认。</Note>
