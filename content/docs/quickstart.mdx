NebulaClaw **auto-creates and hosts your Telegram Bot** — you never need to touch BotFather. Here's the flow:

<Steps>
  <Step title="Sign in">
    Go to the [NebulaClaw Console](https://app.nebulaclaw.ai/login) and sign in with your **Google account**. A workspace is created automatically on first login.
  </Step>

  <Step title="Subscribe ($9.99/month)">
    Creating a Bot requires a monthly subscription:

    - **$9.99 / month** (auto-renews, cancel anytime)
    - **$5.00 monthly token credits** included (unused balance **resets to zero** at the end of each billing cycle)
    - Need more? Buy **top-up packs** from Billing — top-up credits **never expire**

    <Tip>Click **Get Started** to check out via Stripe. You're returned to the creation wizard on success.</Tip>
  </Step>

  <Step title="Pick a community preset">
    Choose the preset closest to your community:

    | Preset | Best for |
    |--------|----------|
    | **Project Community** | Web3 projects, product communities — Q&A + anti-shill |
    | **KOL / Alpha Group** | Paid alpha / research groups — signal-to-noise control, DYOR reminders |
    | **Trader Community** | Trading discussion — neutral, no investment advice |
    | **Meme Community** | Degen / meme-coin groups — high-energy vibes + strict scam-link enforcement |

    Each preset ships with a matching persona prompt. You can further **customize** it with project-specific instructions.
  </Step>

  <Step title="Pick a model tier">
    Three tiers to choose from:

    | Tier | Example models | Best for |
    |------|---------------|----------|
    | **Economy** | `gpt-4.1-nano`, `gpt-4o-mini` | High volume, cost-sensitive |
    | **Balanced** (recommended) | `gpt-5-mini`, `gpt-4.1-mini` | General use, balanced quality and cost |
    | **Premium** | `gpt-5.2`, `gpt-4o`, `gpt-4.1` | Demanding Q&A, complex moderation |

    <Note>Default is Balanced with `gpt-5-mini`. You can switch the model later in Agent settings.</Note>
  </Step>

  <Step title="Name and create the Bot">
    Give the Bot a slug (lowercase letters, numbers, hyphens — max 63 chars), e.g. `my-project-bot`.

    <Frame>
      <img src="/images/create-bot/03-name-bot.png" alt="Create wizard · Step 3 Name Bot" />
    </Frame>

    Click **Create Bot**. The platform automatically:

    1. Assigns and hosts a Telegram Bot from the Bot Pool
    2. Returns a `@botUsername` you can use in Telegram immediately
    3. Boots the Agent gateway — ready in about 30 seconds

    On success you'll see a confirmation page with your **Bot's `@username`**, the **Telegram deep link**, and the next 3 onboarding steps:

    <Frame>
      <img src="/images/create-bot/04-bot-created.png" alt="Bot created successfully — shows @botUsername and next steps" />
    </Frame>

    <Tip>Click the blue `t.me/<botUsername>` link to jump straight to the Bot chat.</Tip>
    <Warning>**The Bot's avatar and username cannot be changed after creation.**</Warning>
  </Step>

  <Step title="Add the Bot to your group">
    1. In your Telegram group → Add members → search `@botUsername` and add it
    2. **Promote the Bot to group admin** (required for moderation)
    3. In the console, go to **Moderation → Telegram Groups** and **add the group ID**

    ### Get the group ID with @userinfobot (3 steps)

    **No need to add @userinfobot to the group** — you query it from the private chat.

    <Steps>
      <Step title="Open @userinfobot and tap Menu → Group / My Group">
        Search [@userinfobot](https://t.me/userinfobot) on Telegram → tap **Start** → tap **Menu** to open the 9-tile menu:

        - **Group** = all groups where you're an admin
        - **My Group** = groups you created

        <Frame>
          <img src="/images/chat-id/01-menu-grid.png" alt="@userinfobot 9-tile menu with Group / My Group entries" />
        </Frame>
      </Step>
      <Step title="Select the target group from the list">
        A **Choose a Group** view appears. Tap the group you want to connect (note: **the group must have Topics disabled** — a requirement of @userinfobot).

        <Frame>
          <img src="/images/chat-id/02-choose-group.png" alt="Choose a Group list — pick the target" />
        </Frame>
      </Step>
      <Step title="Copy the Chat ID into the console">
        The bot replies with `Id: -100xxxxxxxxx`. Copy this ID into **Moderation → Telegram Groups** in the console.

        <Frame>
          <img src="/images/chat-id/03-chat-id-result.png" alt="Chat ID returned by @userinfobot" />
        </Frame>
      </Step>
    </Steps>

    <Tip>The same menu also retrieves **Channel / Forum** IDs.</Tip>
  </Step>

  <Step title="Upload knowledge">
    Go to **Knowledge** and upload anything you want the Agent to answer from:

    - Product / project FAQ
    - Whitepaper / doc summaries
    - Community rules
    - Troubleshooting notes

    Content is semantically indexed — users can ask "how do I withdraw?" or "what's the fee?" and the Agent will answer even without keyword hits.

    <Tip>The persona prompt controls *how* the Agent speaks; the knowledge base controls *what* it says. Putting content in the knowledge base is more effective than stuffing it into the prompt.</Tip>
  </Step>

  <Step title="Set up moderation rules">
    Go to **Moderation → Rules** and click **Add Rule**. Each rule has:

    - **Description** (natural language) — one sentence describing the trigger
    - **Action**: `mute` (duration or permanent) / `kick` (can rejoin) / `ban` (permanent)

    **Recommended baseline ruleset** (copy-paste ready):

    | Trigger | Action |
    |---------|--------|
    | Insults, personal attacks, continued provocation, or defamation — including against the project, team, or community members | `kick` |
    | Scams or phishing messages | `kick` |
    | Requesting or disclosing private keys / seed phrases | `ban` |
    | Spam / advertising | `mute 24h` |
    | Malicious links | `mute 1h` |
    | Fake airdrops, referral-farming schemes | `mute 24h` |

    The AI judges by context — no keyword lists needed. Every action is logged in **Audit** for review.
  </Step>

  <Step title="Check your analytics">
    **Analytics → Dashboard**:

    - Daily / weekly message volume
    - Active users
    - Sentiment trend
    - Moderation action counts

    **Billing → Usage** shows current-month token consumption and remaining credits.
  </Step>
</Steps>

## FAQ

<AccordionGroup>
  <Accordion title="Why isn't my Bot responding?">
    Check two things: ① The Bot must be a group admin; ② The group ID must be added in **Moderation → Telegram Groups**. Either missing and the Agent won't respond in that group.
  </Accordion>
  <Accordion title="Moderation false-positives hitting normal users">
    Go to **Moderation → Rules** and make the description more specific. For example, refine "promotional external links" to "invite links to non-whitelisted domains". The AI calibrates to your description.
  </Accordion>
  <Accordion title="How do I make the Agent more accurate?">
    Expand the knowledge base first, then tune the persona. Content in the knowledge base beats bloating the persona prompt.
  </Accordion>
  <Accordion title="What happens if token credits run out?">
    When the monthly $5 runs out, the bot draws from your top-up balance automatically. If both are depleted, the bot pauses. Buy a top-up pack to resume — **top-up credits never expire**. Monthly credits refresh to $5 at the start of the next billing cycle (unused balance **does not carry over**).
  </Accordion>
  <Accordion title="Can I run multiple Bots?">
    Currently **one Bot per account**. But a single Bot can **serve multiple Telegram groups** — add the same `@botUsername` to each group as admin, then register each Chat ID in **Moderation → Telegram Groups**.

    **Multi-Bot** (multiple independent Bots per account) is on the roadmap — announcements will ship here and on official channels.
  </Accordion>
</AccordionGroup>
