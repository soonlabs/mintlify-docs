The NebulaClaw API lets you programmatically manage agents, moderation rules, knowledge bases, and more.

## Base URL

```
https://api.nebulaclaw.ai/api/v1
```

## Response Format

All responses follow a standard format:

```json
{
  "code": 0,
  "message": "OK",
  "data": { ... }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `code` | number | `0` for success, non-zero for errors |
| `message` | string | Human-readable status message |
| `data` | any | Response payload (null on error) |

## Error Codes

| Code | Meaning |
|------|---------|
| `0` | Success |
| `61` | Unauthorized — invalid or expired token |
| `4xx` | Client error — check request parameters |
| `5xx` | Server error — retry later |

## Content Type

All requests and responses use `application/json`.

```bash
Content-Type: application/json
```

## Rate Limiting

API requests are rate-limited per account. If you exceed the limit, you'll receive a `429 Too Many Requests` response. Wait and retry with exponential backoff.
