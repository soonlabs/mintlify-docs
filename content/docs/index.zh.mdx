**NebulaClaw 是一个 AI 驱动的社区自动化平台。** 它用一个理解自然语言、具备上下文感知能力的 AI Agent，替代传统群机器人依赖 `/命令` 和关键词正则的管理方式。

社区运营的三件核心事务——**回答问题、审核发言、运营洞察**——都交给 Agent 自动完成。配置只需自然语言描述，5 分钟即可部署上线。

## 技术特征

- **自动部署 Bot**——付费后系统自动创建并托管 Telegram Bot，用户无需与 BotFather 交互，创建完成即可直接拉群使用。
- **语义审核引擎**——规则用自然语言描述（例如"用户发布推广外链"），AI 结合上下文判断，显著低于关键词匹配的误杀率。
- **多语言自动识别**——自动检测用户消息语种并用对应语言回复；支持回退语言配置（英、中、韩、土、印尼、俄、越）。
- **$9.99/月订阅**——每月自带 $5 Token 额度（每个计费周期结束时清零）；额外需求可购买充值包，**充值额度永不过期**。

## 解决的企业痛点

从项目方 / 运营团队角度，NebulaClaw 针对以下六个维度的常见痛点：

<CardGroup cols={2}>
  <Card title="人力成本" icon="user-minus">
    **痛点**：7×24 社区值守需要 3 班倒的社区经理，Web3 / 跨境项目的多语言覆盖又会乘上倍数；节假日、深夜恰恰是诈骗链接最活跃的时段。

    **解决**：一个 Agent 替代值守层——自动回答、自动审核、永不下线；团队只需关注 Agent 拦不住的真正高价值对话。
  </Card>
  <Card title="响应时效" icon="timer">
    **痛点**：一条空投钓鱼 / 假返利的消息，在大群里 30 秒内就能完成诱骗；人工 Mod 的平均响应窗口在分钟级，等到介入时损失已经发生。

    **解决**：Agent 实时识别并处置，从消息发出到 mute / kick / ban 在**秒级**完成，不给钓鱼消息扩散窗口。
  </Card>
  <Card title="规则可维护性" icon="wrench">
    **痛点**：关键词 / 正则词库需要持续更新，黑产每换一批域名、一套话术就得补规则；漏过了误杀了都是信任损失。

    **解决**：用**自然语言**描述意图（例如"非白名单邀请链接"、"假空投返利话术"），AI 结合上下文判断，词库不再需要维护，描述写得越清晰执行越精准。
  </Card>
  <Card title="跨语言覆盖" icon="languages">
    **痛点**：Web3 / 跨境社区自然同时出现英、中、韩、土、印、俄、越等语种，按语言分群成本高，合并管理又会出现"看不懂导致误判"。

    **解决**：**自动识别 7 种语言**，Agent 按用户语言回复，审核规则同样跨语言生效；不需要分群、不需要分别训练词库。
  </Card>
  <Card title="合规与审计" icon="scroll-text">
    **痛点**：人工 Mod 标准不统一，同类违规不同处置，后续申诉 / 维权时缺乏证据链；DAO / 上市公司级的社区尤其敏感。

    **解决**：所有处置动作进入 **Audit 日志**，包含触发规则、上下文、动作类型和时间戳；规则统一、处置可回查，满足内审与合规诉求。
  </Card>
  <Card title="运营洞察" icon="chart-column">
    **痛点**：传统 Bot 只做动作不留数据，管理层看不到社区健康度，KOL 合作、活动复盘全凭感觉。

    **解决**：内置**数据看板**——消息量、活跃用户、情绪趋势、审核动作量化输出，把社区从"黑箱氛围"变成可衡量资产。
  </Card>
</CardGroup>

<Card title="加分项 · 部署与扩展" icon="rocket">

## 适用场景

- **Web3 项目社区**——白皮书答疑、反 shill、反诈骗链接过滤
- **KOL / Alpha 付费群**——内容氛围维护 + 噪音过滤
- **交易员社群**——市场讨论、合规提示（不提供投资建议）
- **Meme / Degen 社群**——高能量氛围 + 严格的安全把关（诈骗链接、假空投）
